Descripcion de conceptos

  1. URL: Uniform Resorce Locator (Localizador Uniforme de Recursos) Es la dirección de un sitio en internet, puede escribir a direccion de un sitio especifico o parte de la informacion del mismo.
  2. MIME: Multipurpose Internet Mail Extensions, describe cualquier tipo de contenido y lo manda a traves de la red. Estos son tipos de datos como texto, imagen, audio que los archivos contienen, pueden tener mas de uno, estos son intercambiados de manera transparente.
  3. Metodos y códigos de errores HTTP: sobre url
  4. HTML5: HyperText Markup Language como sus siglas lo indican es la quinta versión del lenguaje HTML el cual funciona como un estándar para definir la estructura y el contenido de una página web.
  5. JS:Java Script, es un lenguaje de programacion el cual esta integrado en las paginas web y utiliza el paradigma orientado a objetos.
  6. PHP: Hypertext Preprocessor, es un lenguaje famoso que se especifica en cubrir el desarrollo de la web y puede ser embebido en HTML.
  7. JSP: Java Server Pages, es una tecnologia encaminada a la creacion de paginas web por medio de programacion en Java. Con esta se pueden crear aplicaciones web que pueden ser ejecutadas en diferentes servidores.
  8. Ajax: Tecnica para el desarrollo de aplicaciones web, utiliza javascript y XML asincrono. Le permite a una pagina wb solicitar nueva informacion del servidor.

**¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
Método: Propósito:
GET: Solicita la representacion de un recurso, recupera datos unicamente.
Post: Envia un recurso especifico a una entidad pero causa regularmente un cambio en el estado o efectos secundarios en el servidor. Por medio de este se comunica el navegador con el servidor.
PUT: Reemplaza los valores actuales del recurso, dependiendo de la peticion y el destino que se halla solicitado.
PATCH: Es utilizado para modificar parcialmente los elementos dentro del recurso.
DELETE: Borra un recurso en especifico

**¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No es responsabilidad del programador pues el error 404 tiene dos interpretaciones la primera es desde el lado del usuario, el link fue ingresado de manera incorrecta, el url no existe y/o tiene un formato no valido. La segunda es del lado del programador en el area de desarrollo de aplicaciones web en esta si seria su responsabilidad cambiarla porue fue oculta por seguridad.

**¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500 porque este es un error del servidor, donde el error puede encontrarse en alguna linea del codigo y si el servidor se equivoca es por el desarrollador, ademas el servidor no puede generar el codigo HTML para devolver al usuario por lo que se imprime el error 505.

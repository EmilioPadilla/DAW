Descripción de conceptos

  1. URL: Uniform Resource Locator (Localizador Uniforme de Recursos) Es la dirección de un sitio en internet, puede escribir a dirección de un sitio específico o parte de la información del mismo.
  2. MIME: Multipurpose Internet Mail Extensions, describe cualquier tipo de contenido y lo manda a través de la red. Estos son tipos de datos como texto, imagen, audio que los archivos contienen, pueden tener más de uno, estos son intercambiados de manera transparente.
  3. Métodos y códigos de errores HTTP: Los métodos HTTP permiten que el servidor sepa lo que se quiere hacer por medio de una url, los más importantes son POST,GET,PUT,DELETE y HEAD. Los códigos de errores son lanzados una vez el servidor reciba una petición estos en general indican si la solicitud se cumplio o no.
  4. HTML5: HyperText Markup Language como sus siglas lo indican es la quinta versión del lenguaje HTML el cual funciona como un estándar para definir la estructura y el contenido de una página web.
  5. JS:Java Script, es un lenguaje de programación el cual está integrado en las páginas web y utiliza el paradigma orientado a objetos.
  6. PHP: Hypertext Preprocessor, es un lenguaje famoso que se especifica en cubrir el desarrollo de la web y puede ser embebido en HTML.
  7. JSP: Java Server Pages, es una tecnología encaminada a la creación de páginas web por medio de programación en Java. Con esta se pueden crear aplicaciones web que pueden ser ejecutadas en diferentes servidores.
  8. Ajax: Técnica para el desarrollo de aplicaciones web, utiliza javascript y XML asíncrono. Le permite a una página web solicitar nueva información del servidor.

Preguntas a Responder:

**¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es el sistema que conforma la gran red de comunicación mundial, mientras que la World Wide Web es un sistema que funciona a través de internet por medio de este se pueden transmitir diversos tipos de datos por medio del protocolo HTTP, permite la navegacion a traves de páginas web. En otras palabras la internet es la base para la comunicación y puede usar varios protocolos mientras que la www se usa para acceder a internet y contiene enlaces que permiten visualizar información por medio del protocolo HTTP.

**¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
Método: Propósito:
GET: Solicita la representación de un recurso, recupera datos únicamente.
Post: Envia un recurso específico a una entidad pero causa regularmente un cambio en el estado o efectos secundarios en el servidor. Por medio de este se comunica el navegador con el servidor.
PUT: Reemplaza los valores actuales del recurso, dependiendo de la petición y el destino que se haya solicitado.
PATCH: Es utilizado para modificar parcialmente los elementos dentro del recurso.
DELETE: Borra un recurso en específico
**¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método que se utiliza para enviar un formulario HTML es el método POST dado que este es mas seguro ya que la información como la de las contraseñas se oculta y el usuario no puede verlos.

**¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET porque este método se encarga de obtener información y traer los datos almacenados en el servidor para así poder acceder a una página a través de un URL.

**Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No es un error HTTP, de hecho significa que la pagina fue cargada de forma correcta, la petición llego correctamente

**¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No es responsabilidad del programador pues el error 404 tiene dos interpretaciones la primera es desde el lado del usuario, el link fue ingresado de manera incorrecta, el url no existe y/o tiene un formato no válido. La segunda es del lado del programador en el área de desarrollo de aplicaciones web en esta si seria su responsabilidad cambiarla porque fue oculta por seguridad.

**¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500 porque este es un error del servidor, donde el error puede encontrarse en alguna linea del codigo y si el servidor se equivoca es por el desarrollador, además el servidor no puede generar el código HTML para devolver al usuario por lo que se imprime el error 505.

    Sobre el ciclo de vida y desarrollo de los sistemas de información:

    **¿Cuál es el ciclo de vida de los sistemas de información?

    El ciclo de vida de un sistema de información se conforma de 7 etapas. Este facilita el control de operaciones del proyecto y del proceso que este sigue a lo largo de su vida.

    Fases:

  • Planeación
  • Análisis
  • Diseño
  • Implementación
  • Mantenimiento

    • **¿Cuál es el ciclo de desarrollo de sistemas de información?

    El ciclo de vida de desarrollo de un sistema es una serie de pasos o etapas de la ingenieria de software para el desarrollo del proyecto/producto de software que se desea realizar. Hay diferentes tipos de modelos que pueden suplir el desarrollo pero estas son las etapas generales.

    Estapas:

  • Planeacion: Definicion de los requerimiendos y funcionalidades del software. Analisis de riesgos y se definen los requisitos.
  • Definicion de los requerimientos: Documentacion de los requisitos especificos.
  • Diseño: Arquitectura del software, documentacion del diseño que son analizadas por la gerencia para saber cual les conviene mas.
  • Análisis del sistema: os desarrolladores trazan el plan, limitaciones, cambios, posibilidades y recursos.
  • Diseño de Software: Diseño del producto basado en los requisitos y el analisis, el diseño fisico y logico.
  • Codificación:Implementacion del diseñp de software, la parte de programacion.
  • Pruebas: La evaluacion del desarrollo de software.
  • Implementación:Instalacion del software en las maquinas de los clientes.
  • Mantenimiento y Funcionamiento: Confirmar el funcionamiento del software en terminos de eficiencia. Se pueden prosentar problemas y/o retos de virus ocultos o problemas en general.
  • Disposición: Archivar datos y componentes de software en caso de que falle su ejecucion.
